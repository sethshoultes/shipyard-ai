# The 3 Things That Matter

## 1. Simplicity is the interface

A PRD drops into GitHub. Three things happen in sequence. A URL comments back. Everything else is distraction. The entire value lives in that loop — nothing external, nothing orchestrated, nothing clever. If you add a queue, a dashboard, a backend service, you've already lost. The moment it requires a person to click through a form or wait for status, you've broken the promise.

## 2. Design system is non-negotiable

Every site is beautiful the moment it exists. Not passable. Not acceptable. Beautiful. This isn't negotiable because it's your only defense against "AI-generated garbage." The typography, color, spacing, imagery strategy — all of it is locked in lookup tables before a single seed is generated. Taste is baked in. If it doesn't look like something Steve would ship, it does not go live.

## 3. Every failure must speak

The moment something breaks, the GitHub issue knows exactly why. No silent failures. No lost jobs. No confusion. The failure is as clear as the success — labeled, logged, commented back. When someone's PRD dies, they understand precisely what went wrong and how to fix it. Communication failure is the death of trust.

---

A PRD written in prose becomes a live website in 90 seconds with zero waiting, zero fussing, zero doubt about quality.
