## What Agency Should Do Differently

**Invert documentation ratio**
- 80% shipping, 20% documenting (not inverse)
- Ship code first, then write retrospective
- Deliverables folder should have build artifacts, not just markdown

**Lead with synthesis**
- Line 1: verdict and rationale
- Lines 2-20: action plan
- Everything else: appendix for deep divers

**Enforce Jensen's Rule on intake**
- Every issue must answer: Can AI do this automatically?
- Reject infrastructure-only work without intelligence layer
- Build meta-solutions, not instances

**Match process weight to work complexity**
- 1-line fix = 1-page plan + ship
- Multi-week feature = full Great Minds treatment
- Don't use sledgehammer on thumbtack

**Show, don't tell**
- Screenshots required
- Before/after diffs required
- Working demo URL required
- Empty deliverables = incomplete work

**Unify voice across artifacts**
- Choose register once, maintain throughout
- Either personality-driven OR corporate, not mixed
- Consistency creates quiet power

**Build platform value from primitives**
- This binding → AI Security Fabric opportunity
- One fix → automated fleet-wide monitoring
- Instance work should unlock meta-work

---

## Key Learning

When the fix takes 3 minutes but the debate takes 3 days, you're optimizing the wrong variable — ship first, then decide if it deserves philosophy.
