# Steve — Round 2: Promptfolio

Elon is optimizing for the wrong metric: engineering velocity
over user dignity. "Ship structure first, aesthetic polish second"
is exactly how you build tools that nobody loves. A portfolio that
looks like a CRUD app is a resume in Comic Sans — technically
functional, spiritually dead.

He wants a "Made with Promptfolio" footer link as a distribution
lever. That's a billboard in the Louvre. Users aren't billboards.
Viral coefficient is a spreadsheet fantasy; what spreads is awe.
You don't growth-hack pride.

Where I concede: kill the "Try this prompt" widget. I never asked
for a chat UI, and Elon's right — it's a second startup disguised
as a feature. Claude-only JSON import for v1 is pragmatic; elegance
includes knowing what to defer. Built-in analytics should be a
one-line embed, not a dashboard. And yes, stream-parse that JSON
so shared hosting doesn't choke.

But defend this: design isn't polish here — design is the entire
value proposition. Promptfolio turns a chat log into a *body of
work*. If the frame doesn't elevate the art, there is no product.
One exquisite template isn't a limitation; it's curation. The user
doesn't want a website builder. They want their craft to look
inevitable.

## Non-Negotiables

1. **Zero-config first 30 seconds.** JSON drops in; a hero card
lives. No wizard, no layout picker, no "choose your industry."
One perfect default.

2. **Single, sacred template.** No theme builder, no custom CSS
panel, no forty-seven variations. We ship one gallery wall that
feels inevitable.

3. **No adware footer.** No "Made with Promptfolio" link polluting
the user's page. Their work gets the final pixel. Distribution
happens because the portfolio is breathtaking, not because we
graffiti it.

Build it in four hours or four days — I don't care. Build it right.
