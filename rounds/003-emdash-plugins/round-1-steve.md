# Steve Jobs — Round 1 Positions

## Product Naming

**MemberShip** is clever. I like the capitalization play. It says exactly what it is while having personality. Keep it.

**EventDash** is a disaster. It sounds like a cheap analytics tool from 2012. An event isn't a "dash" — it's a *moment*, a *gathering*, a *reason to show up*. Call it **Convene**. One word. Evokes bringing people together. Or **Rally**. Even **Gather** is better than this forgettable hyphenate.

## Design Philosophy

The quality bar in this PRD mentions TypeScript and tests. That's table stakes. Where's the *soul*?

What makes this insanely great is **invisible complexity**. A yoga studio owner should create an event in fewer taps than ordering a latte. The Stripe integration shouldn't feel like "integration" — it should feel like money just *appears* when someone registers.

Block Kit admin UI is the right foundation. But don't just use it — *disappear into it*. The plugin should feel native to EmDash, not bolted on. When a user installs this, it should feel like EmDash was incomplete without it.

## User Experience — The First 30 Seconds

Install. One command. The plugin registers itself.

The user opens their admin. A single new section appears: "Events" or "Members." Not both screaming for attention.

They click. An empty state that doesn't make them feel empty — it makes them feel *excited*. "Your first event is 60 seconds away." One button. One form. One publish.

That's it. If setup takes longer than making coffee, we've failed.

## Brand Voice

EmDash is sophisticated simplicity. These plugins must speak the same language.

- Not "Configure your Stripe webhook endpoint" — "Connect Stripe"
- Not "Registration confirmation email template" — "What should we say when someone signs up?"
- Not "Granular access levels" — "Who gets in?"

Conversational. Confident. Never condescending. We explain what things *do*, not how they work.

## What to Say NO To

**NO** to event categories and tags in v1. Complexity creep. Let content live and breathe first.

**NO** to multiple payment processors. Stripe or nothing. Simplicity is choosing, not accommodating.

**NO** to recurring event patterns in v1. Get single events perfect. Recurrence is a v2 problem.

**NO** to custom email templates with WYSIWYG editors. Ship with beautiful defaults. Let power users edit code if they must.

**NO** to attendee comments and discussion threads. This is not a social network. Ship focused tools.

## The Emotional Hook

People will love this because it removes *dread*.

Small business owners dread membership software. They dread event management. They've been burned by tools that promise simplicity and deliver spreadsheets wearing makeup.

These plugins say: "You opened a yoga studio to teach yoga. You opened a bakery to bake. We'll handle the rest."

That's not a feature. That's *freedom*. And freedom is worth paying for.
