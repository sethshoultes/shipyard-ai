# Round 2: Steve

Elon wants to ship a utility. I want to ship a seduction.

## Where Elon Optimizes for the Wrong Metric

**Engineering elegance over user lust.** "200 lines of code" is a vanity metric. Users don't share your line count; they share the feeling in their chest. Cutting Claude to save $0.02 and 3 seconds? That transforms a miracle into a mail merge. `repo.description` is exactly what developers wrote when they were half-asleep. If the poster reads like a placeholder, it *is* a placeholder.

**Async generation with a placeholder.** Elon says return a placeholder on cache miss. A placeholder is a broken promise. The "gasp" moment requires instantaneity. Paste, hold breath, gasp — not paste, hold breath, wait, receive a gray box. That kills the viral loop before it starts.

## Defending What Elon Would Cut

Elon calls PNG-only "arbitrary." It isn't. JPG compresses text into dirt. When a developer sees their repo name rendered with artifacts, the magic dies. PNG is crisp, honest, perfect.

"No customization" is not stubbornness. Customization panels are where good products go to die. Users are not designers. Give them sliders, and they will make ugly things — then blame you. We give them one inevitable layout because we have taste and they hired us for it.

## Where Elon Is Right

Caching. Pre-warming trending repos. Solving GitHub rate limits on day one. These are table stakes. I concede the infrastructure must not buckle. And yes, GitHub Actions is v2. The "paste a URL" simplicity is sacred.

## Non-Negotiables

1. **The name is Poster.** One word. No "Child." No apology.
2. **One inevitable layout.** Not a template. Not a choice. The only way it could look.
3. **The 30-second gasp.** No placeholders. No `repo.description` shortcuts. If the first render isn't breathtaking, we haven't built a product — we've built a utility.
