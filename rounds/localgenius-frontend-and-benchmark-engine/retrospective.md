# Retrospective — SPARK

## What Worked
- Debate format forced clarity. Elon cut bloat. Steve defended soul. Phil found third way.
- Scope lock was ruthless. 11 decisions killed more than they kept.
- Backend architecture solid. KV worker, D1 cron, suppression logic — functional.
- Name decision fast. SPARK won in one round. No committee rot.
- Demo script excellent. Two minutes of story. Pity it was only words.

## What Failed
- Built engine. Forgot driver. `get_benchmarks()` lives in API. Never called in admin.
- Demo page missing. KV worker ready. No HTML. Opening scene absent.
- QA blocked twice on placeholder content. Same line, two passes. Shame.
- Weekly digest: specified, not built. Retention hook became air.
- Board verdict: HOLD at 4/10. "All engine, no emotion."
- One-agent-session rule broken. Scope exceeded capacity. Again.

## Do Differently
- Wire UI before building API. Backend without frontend is a cave without entrance.
- Surface the soul immediately. If benchmark is the product, burying it is self-defeat.
- QA earlier. Placeholders are not edge cases. They are negligence.
- Cut until one agent can finish in four hours, then cut once more.
- Emotional beats are not v1.1. They are v1 or nothing.

## Key Learning
- A backend no user sees is a moat dug in fog.

## Process Adherence: 5/10
- Rituals observed. Debate, decisions, review — all present.
- Substance ignored. Locked scope exceeded. Emotional hooks buried. QA failed.
- Strong process. Weak obedience.
