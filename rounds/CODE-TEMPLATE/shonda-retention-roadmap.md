# Shonda Rhimes Retention Roadmap — CODE-TEMPLATE v1.1

**Thesis:** A template nobody opens twice is a script nobody watches twice. Retention is not about function; it is about *feeling*. If the builder does not see themselves as the protagonist of their own efficiency story, they will abandon the show.

---

## What Keeps Users Coming Back

### 1. The Progress Loop (Narcissism as Fuel)
Builders return when the mirror gets better every time.

- **Validation Score:** A 0-100 quality grade on every PRD save, surfaced like a credit score. No one ignores a number attached to their name.
- **Improvement Graph:** A sparkline showing PRD quality over the last 10 drafts. Momentum is visual addiction.
- **Streak Counter:** "17 days of green gates." Break the streak and the template dares you to start again.
- **Personal Best Alerts:** Push-style notifications when a user beats their own clarity score.

### 2. The Builder Persona Journey (Story Arc)
Drop the medical-chart spec dump. Write the user into a five-act structure:

| Day | Beat | Emotional State |
|-----|------|----------------|
| 1 | **Hallucination Hell** — User pastes a broken prompt into an agent. Chaos. | Shame / frustration |
| 2 | **The Diagnosis** — Template shows exactly why the build failed. | Relief / curiosity |
| 3 | **First Green Gate** — Validator passes. Agent executes verbatim. | Power / competence |
| 5 | **The Team Notices** — Colleague asks, "How did your build not break?" | Pride / status |
| 7 | **They Ask for YOUR Template** — User becomes the internal standard. | Ownership / mastery |

Every new user should see this arc in the onboarding modal, with their own name written into Day 1.

### 3. Episodic Case Studies (Content Flywheel)
Turn the before/after examples into *aired episodes* with titles, stakes, and credits.

- **Episode 1:** "The PRD That Saved AgentPress" — How one spec prevented a $4,000 rebuild.
- **Episode 2:** "The Verbatim Contract" — When the agent tried to improvise and the gate said no.
- **Episode 3:** "The Designer Who Couldn’t Code" — A Figma-native PM writes her first green-gate PRD.

Each episode ends with a cliffhanger teaser for the next, plus a CTA: *Submit your own PRD for Season 2.* Users generate the content that keeps other users watching.

### 4. Stakes at Every Exit
End every section with consequence, not maintenance manuals.

- **Skip verbatim contracts → watch build burn.**
- **Skip background context → agent hallucinates your user model.**
- **Skip hollow-build gate → 47-minute rebuild at 2 a.m.**

Fear is a hook when it is specific. Vague warnings are fast-forwarded. Exact horror stories are remembered.

### 5. Social Proof & Leaderboard
Builders are competitive. Use it.

- **Spec Leaderboard:** Highest clarity scores this week, most gates passed, longest streaks.
- **Template Authorship Credit:** When a team adopts the tool, the originating user gets a "Created by [Name]" badge on the repo README.
- **Cross-Team Visibility:** Slack/Discord integration that broadcasts green-gate celebrations (opt-in, dopamine-forward).

## v1.1 Features (Retention-First)

### Narrative & Onboarding
- [ ] **Prologue Mode:** First-run experience that asks the user their role (Engineer, PM, Designer, Writer) and replays the Day 1→Day 7 arc with role-specific subtitles.
- [ ] **The Diagnosis Engine:** Not just pass/fail. Explain *why* the gate failed in plain language: "Your acceptance criteria are a wish list, not a contract. Here is the rewrite."
- [ ] **Voice Toggle:** Switch between "Clinical," "Coach," and "Drama" validator tone. Some users want a drill sergeant; others want a showrunner.

### Progress & Gamification
- [ ] **PRD Pulse Dashboard:** Personal landing page with streak, score history, time-saved estimate, and "episodes watched" progress.
- [ ] **Season Badges:** Quarterly achievements — "Survived Scope Creep," "Verbatim Victor," "Zero-Hollow Hero."
- [ ] **Comeback Trail:** If a user is absent 14 days, trigger a single-case-study email with the subject line: *The spec that almost died. (Yours didn’t have to.)*

### Content & Community
- [ ] **User-Submitted Episodes:** Portal to submit before/after PRDs. Curated monthly. Credit the author.
- [ ] **The Writer’s Room:** Template section for "Spec Bibles" — best PRDs from real builds, annotated like director’s commentary.
- [ ] **Cliffhanger Release Notes:** End every changelog with a teaser: *Next drop: the feature that makes your agent read your mind. (It can’t yet. Here is the PRD.)*

### Semantic Intelligence (The Plot Twist)
- [ ] **Ambiguity Detector:** LLM-powered "show, don’t tell" check. Flags vague verbs ("handle," "manage," "support") and suggests concrete replacements.
- [ ] **Auto-Acceptance Generator:** Reads the Background section and proposes three testable criteria the user probably forgot.
- [ ] **Spec Similarity Match:** Before writing, surface past PRDs from the team corpus that solved a parallel problem. *Steal from your own writers' room.*

## The 7-Day Retention Metric

**North Star:** % of new users who voluntarily open the template on Day 7 without a failed-gate forcing them to.

| Milestone | Target | Lever |
|-----------|--------|-------|
| Day 1 | 100% onboarded to Prologue | Story arc |
| Day 2 | 60% run first validation | Diagnosis engine |
| Day 3 | 40% achieve first green gate | Stake-driven urgency |
| Day 5 | 25% share score or streak | Social proof |
| Day 7 | 15% return voluntarily | Content flywheel + progress loop |

If Day 7 voluntary return is below 15%, the story is broken. Rewrite the episode, not the ads.

---

*Bottom line: Give builders a reason to open the template tomorrow that has nothing to do with punishment. Give them a mirror, a score, a season finale, and a sequel tease. That is retention.*
