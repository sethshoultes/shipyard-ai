# Board Votes — Dream Cycle 2026-04-25

Six ideas on the table. Five votes. Plurality wins.

---

## Jony Ive — Vote: ScrollTale

**Justification:**
1. Craft is not a luxury; it is the signal that separates professionals from amateurs.
2. WordPress powers 43% of the web, yet reading on it feels like 2003. That gap is shameful.
3. ScrollTale doesn't ask writers to learn a new tool — it elevates what they already create.
4. The best products are those that make users feel proud of their own output.
5. We should build something that raises the baseline of beauty on the internet, one post at a time.

---

## Reed Hastings — Vote: CommandBar

**Justification:**
1. Speed is the only feature that never goes out of style, and CommandBar makes WordPress feel instant.
2. Every wp-admin user wastes minutes per day clicking through nested menus; this gives them those minutes back.
3. The surface area is tiny — one keyboard shortcut, one modal — which means the UX can be flawless.
4. It turns WordPress from a CMS into an operating system, which is the correct mental model for power users.
5. Low support burden, high daily active usage, and it compounds in value as we add more commands.

---

## Sheryl Sandberg — Vote: AutoSchema

**Justification:**
1. SEO is an $80 billion market, and schema markup is the table stakes that most sites fail to implement correctly.
2. The user need is proven — millions pay for Yoast and RankMath — but nobody has made it truly zero-config.
3. The technical architecture is a pipe: WP plugin calls Worker, Worker returns structured data, plugin injects it. Almost unbreakable.
4. Revenue potential is immediate: freemium on WordPress.org, paid tier for custom schema types and bulk processing.
5. Every site that installs it gets better search rankings, which means retention is built into the value proposition.

---

## Peter Thiel — Vote: StaticSnap

**Justification:**
1. The contrarian question is: what are people still doing the hard way that should be automatic? Hosting WordPress dynamically is that hard way.
2. StaticSnap creates a moat by making WordPress competitive with Jamstack on performance without requiring a rewrite.
3. It attacks a real cost center: managed WordPress hosting margins are obscene, and site owners know it.
4. Zero to one potential: if we can make WP static at the edge, we change the architecture conversation for 810 million sites.
5. Technical feasibility is high because we already own the Workers expertise; this is an application, not an infrastructure project.

---

## Phil Jackson — Vote: CommandBar

**Justification:**
1. The agency ships what it uses, and we live in wp-admin — this is the ultimate dogfood product.
2. Buildable in one session: a React command palette, three REST endpoints, and a keyboard listener. We have done harder in four hours.
3. Distribution is pre-built: WordPress.org plugin directory, and every agency client becomes a user on day one.
4. It compounds: extensible architecture means other plugin authors add commands, making CommandBar more valuable without our effort.
5. The user is crystal clear — developers and power users — and they are the same people who recommend tools to their clients.
