## Idea 1: **AgentBench** — Leaderboard for AI Coding Agents

### The Problem
Everyone's building AI coding agents but nobody knows which one actually works. Benchmarks are synthetic. Real developers need real comparisons.

### The Solution
An open benchmark suite that tests AI agents on actual shipping tasks:
- "Add dark mode to this repo"
- "Fix this security vulnerability"
- "Write tests for untested code"

Agents submit via API. Results show: completion rate, code quality score, time to ship, cost per task. Public leaderboard.

### Market Size
Every AI lab (OpenAI, Anthropic, Google, Meta) wants to prove their agent is best. Every dev tool company needs to benchmark. TAM: $500M+ in developer tooling.

### Technical Feasibility
**HIGH.** We already run multi-agent orchestration. Spin up isolated environments, run agents, measure outputs. Cloudflare Workers for the API, simple scoring rubric.

### Distribution
Hacker News (controversial = viral), AI Twitter, direct outreach to AI labs who want bragging rights.

### One Session Build
API endpoint + 5 benchmark tasks + scoring logic + static leaderboard page. Ship it ugly, iterate fast.

## Idea 2: **WP-to-App** — Turn Any WordPress Site into a Mobile App

### The Problem
500M+ WordPress sites. Most businesses want a mobile app but can't afford native development. Existing solutions are expensive ($300/month+) or garbage.

### Market Size
WordPress powers 43% of the web. Even 0.1% conversion = 500K potential customers. At $29/month = $174M ARR opportunity.

### The Solution
A WordPress plugin + build service that:
- Wraps the site in a performant WebView shell
- Adds native features: push notifications, offline mode, app icon
- One-click submission to App Store / Play Store
- AI generates the app store listing from site content

### Technical Feasibility
**HIGH.** We've built WordPress plugins. Capacitor/Ionic handles the native wrapper. The hard part (App Store submission) is a known problem with existing tooling.

### Distribution
WordPress.org plugin directory (free tier), WP developer communities, agencies who serve SMBs.

### One Session Build
Plugin that generates app config + Capacitor build pipeline + hosted build service on Cloudflare.

## Idea 3: **PromptOps** — Version Control for AI Prompts

### The Problem
Every company using AI has prompts scattered in code, Notion, and people's heads. No versioning, no A/B testing, no rollback. It's 2005-era deployment practices for the most critical part of AI apps.

### Market Size
Every company using LLMs needs this. That's every company. TAM: effectively unlimited — this is infrastructure.

### The Solution
- Git-like CLI for prompt management: `promptops push`, `promptops diff`, `promptops rollback`
- Dashboard showing prompt performance metrics (latency, cost, user ratings)
- A/B testing built in: "Run 10% of traffic on the new prompt"
- Works with any LLM provider via middleware

### Technical Feasibility
**VERY HIGH.** It's a proxy layer + metadata store + simple dashboard. We've built harder things. Cloudflare Workers handles the proxy perfectly.

### Distribution
Dev Twitter, Hacker News ("Heroku for prompts"), direct integration with AI dev tools.

### One Session Build
CLI + Workers proxy + SQLite metadata + minimal dashboard. The MVP is just versioned storage + rollback.
