### Idea 1: Relay — AI Form Handler & Lead Router
**What:** A WordPress plugin that replaces dumb contact forms with an intelligent lead-processing engine. Any HTML form or block form submits to Relay, which stores the entry locally and forwards it to a Cloudflare Worker. The Worker sends the submission to Claude for intent classification (sales inquiry, support request, partnership, spam), extracts urgency and sentiment, then routes it to the right destination: email, Slack webhook, or CRM. A searchable inbox lives inside wp-admin where users filter by category, read AI-generated summaries, and draft contextual replies with one click.

**Market:** Every WordPress site has a contact form. Millions of small businesses lose leads because submissions land in a single overwhelmed inbox. The form-builder market (Gravity Forms, WPForms) is mature but lacks native AI. Relay is the first form handler that thinks.
**Feasibility:** The plugin shell is standard PHP (we ship these in one session). Form capture uses a REST endpoint. The Worker is a stateless Claude proxy with a 200-line routing engine. The admin UI is React injected via wp_enqueue_scripts. One session builds the plugin, the Worker, and the inbox UI.
**Distribution:** WordPress.org plugin repository. Freemium: local storage and basic routing free; AI classification, Slack integration, and CRM webhooks paid. WP.org is a self-sustaining acquisition funnel — every install is a lead.

### Idea 2: Beacon — Real-Time Site Telemetry for WordPress
**What:** A lightweight WordPress plugin that captures Core Web Vitals, PHP error rates, JS exception traces, plugin performance impact, and database slow-query logs. Data streams to a Cloudflare Worker dashboard where agencies and developers view real-time telemetry across all their sites. AI surfaces anomalies: "Contact Form 7 slowed page load by 800ms after the 7.0 update" or "AJAX error rate spiked 400% at 14:00 UTC."

**Market:** WordPress agencies manage 10–500 sites each and currently patch together uptime monitors, Query Monitor, and New Relic. There is no telemetry product priced for indie agencies. The addressable market is every professional WordPress developer.
**Feasibility:** The plugin uses existing WP hooks (wp_footer for Web Vitals, set_error_handler for PHP, window.onerror for JS). The Worker aggregates into D1. The dashboard is a Next.js app with SSE for live telemetry. One session builds the plugin + a single-site dashboard; multi-site aggregation is a follow-up.
**Distribution:** WordPress.org for the plugin; SaaS dashboard invites spread through agency Slack channels and Dev.to. Developer tools spread through GitHub README references.

### Idea 3: Forge — WordPress-to-Emdash Migration Plugin
**What:** A WordPress plugin that exports an entire site — posts, pages, media, menus, users, categories, custom fields — into an Emdash-compatible package. A companion Cloudflare Worker transforms the export: converts WP content to Emdash collections, remaps media URLs to R2, generates an Astro theme scaffold based on the existing site's visual structure, and returns a deploy-ready zip. One click inside wp-admin, five minutes of processing, and the user has a modern stack clone of their site.

**Market:** WordPress performance and security fatigue is real. Emdash (Astro + Cloudflare) offers 100/100 Lighthouse scores and zero maintenance overhead. The migration market from legacy CMS to modern stacks is growing as AI-generated content increases site complexity. We capture users at the moment of platform curiosity.
**Feasibility:** We know both WordPress and Emdash cold. The export logic uses existing WP REST API and XML-RPC patterns. The transformation Worker reuses schemas we have already built. One session builds the export plugin and the core transformation pipeline; polish and edge-case handling come in revision rounds.
**Distribution:** Emdash website, Shipyard client onboarding, Hacker News launch, and Dev.to migration guides. Every migration is a platform win for Emdash and a service upsell for Shipyard.
