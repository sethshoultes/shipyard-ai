## Idea 1: **WP Showcase** — Beautiful Plugin Demo Sites in One Click

### The Problem
WordPress plugin developers ship great code but terrible demo sites. The plugin directory is full of ugly screenshots and broken demo links. First impressions matter.

### The Solution
A WordPress plugin that generates a stunning, responsive demo site for any other plugin. Point it at a plugin slug, it creates:
- Hero section with animated feature highlights
- Interactive demo sandbox (iframe with the plugin active)
- Testimonials aggregated from WP.org reviews
- One-click "Install on my site" deep link
- Automatic dark/light mode

### Why It Delights
Plugin authors get Apple-quality marketing pages without hiring a designer. The demo site IS the distribution channel — it converts visitors into installs.

### Distribution
WordPress.org plugin directory, dev Twitter, r/WordPress, ProductHunt

### Dogfood Factor
We use it to showcase Dash and Pinned. Our own plugins become the case study.

## Idea 2: **Commit Poetry** — Git Commits That Tell Stories

### The Problem
Git history is write-only. Nobody reads commit messages because they're robotic and boring. "fix: updated thing" tells you nothing.

### The Solution
A CLI tool (and GitHub Action) that rewrites commit messages into engaging micro-narratives. Uses AI to understand the diff and writes commits that:
- Explain the *why* not just the *what*
- Include relevant emoji that actually means something
- Link to the mental model behind the change
- Optional: generates a weekly "changelog story" for releases

### Why It Delights
Reading git log becomes enjoyable. New team members understand codebase history. Release notes write themselves.

### Distribution
npm, GitHub Marketplace, dev Twitter, Hacker News

### Dogfood Factor
We run it on shipyard-ai itself. Our commit history becomes a living tutorial.

## Idea 3: **Presence** — Ambient AI Status for Remote Teams

### The Problem
Remote work killed "I can see you're in flow state, I won't interrupt." Slack status is manual and everyone ignores it. We need ambient awareness without surveillance.

### The Solution
A lightweight desktop app + Slack integration that:
- Detects focus state from window activity (no keylogging, just app switches)
- Sets beautiful, expressive statuses: "Deep in code", "Context switching", "Wrapping up"
- Shows a gentle "interrupt cost" indicator (high/medium/low)
- Learns your patterns: "Usually available after standup"

### Why It Delights
It brings back the peripheral awareness of physical offices. Beautiful animations. Respects privacy while enabling connection.

### Distribution
Slack App Directory, remote work communities, ProductHunt

### Dogfood Factor
We use it to coordinate the agency. Shows when agents are "thinking" vs "idle."
