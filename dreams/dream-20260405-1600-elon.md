## Ground Rules I Applied

Before writing a single idea, I stress-tested each against these filters:

1. **Can Shipyard AI build and ship it in one session?** We have proven capability with WordPress/Emdash plugins (EventDash, MemberShip), Cloudflare Workers (contact-form, prd-chat), Astro templates, and full-stack Next.js apps. If it doesn't fit our existing stack muscle, it's out.
2. **Is there a distribution channel that already exists?** Building something nobody can find is the most common failure mode. Every idea must plug into an existing marketplace, directory, or community where buyers are already shopping.
3. **Does it compound?** One-time sales are a treadmill. Recurring revenue or network effects preferred.

## Idea 1: SiteGrade

### One-liner
AI-powered site audit tool that scans any Emdash (or Astro) site and generates a prioritized improvement report with one-click fixes.

### The Market
- **TAM:** ~$4.8B website optimization / SEO tools market (growing 15% YoY)
- **SAM:** The subset building on Astro and Emdash — tens of thousands of sites today, growing fast as the headless CMS movement accelerates. But the audit itself works on any URL, so the real SAM is every small business with a website: ~30M in the US alone.
- **Who pays:** Site owners, freelance web developers, and agencies who manage client sites. Agencies are the whale — they run audits on dozens of sites monthly and will pay for batch tooling.

### Moat
- **Emdash-native intelligence.** Generic auditors (Lighthouse, Ahrefs) don't understand Emdash's schema, plugin system, or template architecture. SiteGrade does. It can say "your EventDash plugin is misconfigured" or "this MemberShip gating rule is leaking content." No competitor can do that without deep Emdash integration, and we are the ones building the plugins.
- **Fix, not just report.** Most audit tools stop at the PDF. SiteGrade generates actual code patches or config changes. This is the leap from diagnostic to therapeutic.

### Distribution
- **Emdash marketplace.** Ship as a plugin/service directly where Emdash users browse. First-mover advantage — there is no audit tool in the Emdash ecosystem yet.
- **Free tier as growth engine.** Let anyone scan one site free. The report is so good they share it. Agencies upgrade for batch scans, scheduled monitoring, and white-label reports.
- **WordPress refugees.** As sites migrate from WordPress to Emdash, they need to validate that their new site is at parity or better. SiteGrade is the migration validation tool.

### Technical Shape
- **Cloudflare Worker** runs the scan engine (performance, accessibility, SEO, schema validation). We already have Workers infrastructure (contact-form, prd-chat).
- **Claude Haiku** generates the natural-language report and prioritized recommendations from raw audit data. Cheap enough to run on every scan.
- **Astro front-end** for the dashboard — we have 8+ Astro templates already. Reuse the marketing template as the shell.
- **Emdash plugin** (optional v1 add-on) that embeds the audit dashboard directly in the Emdash admin panel.
- **Key components:** URL crawler, Lighthouse wrapper (via Cloudflare Browser Rendering API), schema validator (Emdash-specific checks), AI report generator, results DB (D1 or Turso).

### One-Session Scope
**Ships in v1:**
- Single-URL scan via web UI (enter URL, get report)
- Performance, accessibility, SEO, and meta-tag checks
- AI-generated summary with top 5 priorities
- Shareable report link
- Basic Emdash-specific checks (plugin detection, template validation)

**V2 (follow-up sessions):**
- Batch scanning for agencies
- Scheduled weekly monitoring with email alerts
- One-click fix generation (code patches)
- White-label reports
- Emdash admin plugin integration

### Unit Economics
- **Free tier:** 3 scans/month (acquisition)
- **Pro:** $19/month — unlimited scans, scheduled monitoring, PDF export
- **Agency:** $79/month — batch scanning, white-label, API access, team seats
- **Margin:** ~90%+ after Cloudflare/AI costs. A scan costs ~$0.005 in compute + ~$0.002 in Haiku tokens. At $19/month even a light user is >95% margin.

### Why Now
- Emdash ecosystem is early. No audit tools exist. Whoever ships first owns the category.
- Cloudflare Browser Rendering API just went GA — you can run headless Chrome at the edge without managing infrastructure.
- AI report generation makes the output 10x more useful than a raw Lighthouse JSON dump.
- The WordPress-to-headless migration wave is accelerating. Migration validation is an acute pain point with no great tool.

## Idea 2: ShipKit

### One-liner
Open-source CLI + GitHub Action that turns a CLAUDE.md file into a full CI/CD pipeline for any AI-assisted codebase — lint, typecheck, test, deploy — configured in 30 seconds.

### The Market
- **TAM:** ~$15B CI/CD market, but the real angle is the developer tooling layer that's forming around AI-assisted coding.
- **SAM:** Every developer using Claude Code, Cursor, Copilot Workspace, or any AI coding tool. Conservative estimate: 5M+ developers by end of 2026. They all have the same problem: AI writes code fast, but there's no quality gate between "AI generated this" and "this is in production."
- **Who pays:** Individual developers ($9/mo is impulse-buy territory), teams and startups ($49/team/mo for shared configs and dashboards), and enterprises (custom pricing for org-wide policy enforcement).

### Moat
- **CLAUDE.md is our format.** We literally use it. Every Shipyard AI project has one. Every Claude Code user has one (or should). By building the tool that operationalizes CLAUDE.md, we become the standard. Network effects kick in when teams share configs.
- **Dogfooding advantage.** We run this on every project we ship. Our pipeline (intake, debate, plan, build, review, deploy) becomes the reference implementation. Every bug we hit, we fix first.
- **Community flywheel.** Open-source the core CLI. Monetize the hosted dashboard, team features, and GitHub App. Contributors add integrations (Vercel, Cloudflare, Netlify, Railway). Each integration expands the market.

### Distribution
- **npm install -g shipkit.** One command. Developers live in the terminal.
- **GitHub Marketplace.** Ship the GitHub Action on day one. 100M+ repositories on GitHub. Even 0.001% adoption is 100K installs.
- **Claude Code community.** Every Claude Code user is a potential customer. We can submit to the Claude Code extensions/tools ecosystem.
- **Dev Twitter/X + Hacker News.** "We built a CI/CD pipeline generator that reads your CLAUDE.md" is a compelling Show HN. Developer tools go viral when they solve real pain with a great demo.

### Technical Shape
- **Node.js CLI** (TypeScript). Reads CLAUDE.md, package.json, tsconfig, and project structure. Infers the right CI/CD pipeline.
- **Claude Haiku** for intelligent config generation — understands project intent from CLAUDE.md, generates appropriate test strategies and deploy configs.
- **GitHub Action** (YAML template generator + runner). Outputs .github/workflows/shipkit.yml.
- **Hosted dashboard** (Astro + Cloudflare Workers + D1) for team management, run history, and analytics.
- **Key components:** Project analyzer (file system scanner), config generator (templates for major frameworks), GitHub API integration, CLI framework (Commander.js or similar).

### One-Session Scope
**Ships in v1:**
- CLI that reads CLAUDE.md + project files and generates a GitHub Actions workflow
- Support for: Node.js (npm/pnpm), Python, and generic projects
- Commands: `shipkit init` (generate config), `shipkit check` (dry-run locally), `shipkit badge` (add status badge to README)
- Pre-built templates for: lint, typecheck, test, build, deploy (Vercel, Cloudflare Pages)
- Open-source on GitHub with MIT license

**V2 (follow-up sessions):**
- GitHub App with hosted dashboard
- Team shared configurations
- AI-powered test generation from CLAUDE.md descriptions
- Slack/Discord notifications
- Cost tracking (how much AI spend per PR)
- Policy enforcement (block merges that exceed token budgets)

### Unit Economics
- **Free (OSS):** CLI + GitHub Action, forever free. This is the growth engine.
- **Pro:** $9/dev/month — hosted dashboard, run history, analytics, priority support
- **Team:** $49/team/month (up to 10 devs) — shared configs, team dashboard, policy enforcement
- **Margin:** 85%+. Hosting costs are minimal (Cloudflare Workers). AI costs only incurred during `init` (one-time per project).

### Why Now
- AI-assisted coding just crossed the chasm. Every developer is experimenting. But the "last mile" — going from AI-generated code to production — is still manual and error-prone.
- CLAUDE.md is becoming a de facto standard for AI coding context. No tool operationalizes it yet.
- GitHub Actions is the dominant CI/CD platform. Meeting developers where they already are.
- We use this exact workflow daily. The pain is real, the solution is proven, and we can ship v1 from our own muscle memory.

## Idea 3: PluginForge

### One-liner
A hosted platform where you describe an Emdash plugin in plain English and get a production-ready, tested, marketplace-submittable plugin back in minutes.

### The Market
- **TAM:** ~$6.5B low-code/no-code development platform market, but the specific wedge is the WordPress plugin economy ($2B+/year) migrating to headless CMS platforms.
- **SAM:** Every Emdash site owner who needs custom functionality but can't (or won't) write TypeScript. Every freelancer who builds Emdash sites and wants to offer custom plugins without deep coding. Conservatively 50K+ in year one as Emdash grows.
- **Who pays:** Site owners who need a specific feature ($29-99 per plugin generation). Freelancers/agencies who generate plugins for clients ($199/month unlimited). Emdash itself (potential partnership — they want a thriving plugin ecosystem, and we can accelerate it 100x).

### Moat
- **We literally build Emdash plugins.** MemberShip and EventDash are production plugins we built. We know the sandbox API, the Block Kit admin UI, the Astro component model, the email system. This institutional knowledge is baked into the AI prompts. A competitor would need to reverse-engineer all of this.
- **Template library compounds.** Every plugin we generate feeds back into our template library. After 1,000 generated plugins, our system has seen every pattern. The quality gap widens over time.
- **Marketplace lock-in.** If PluginForge-generated plugins become the majority of the Emdash marketplace, we are the infrastructure. Switching costs are zero for individual plugins but massive for the ecosystem dependency.

### Distribution
- **Emdash partnership.** Pitch this as the official plugin generation tool. "Need a plugin? Generate one." Link from the Emdash admin panel directly.
- **Emdash marketplace.** Every generated plugin can be listed (with creator's permission), driving traffic back to PluginForge.
- **Content marketing.** "How to build an Emdash plugin in 60 seconds" — YouTube, Twitter/X, dev blogs. The demo is inherently viral because the before/after is so dramatic.
- **Existing Shipyard AI pipeline.** We already intake PRDs and output plugins. PluginForge is the self-serve version of what we do manually today. Our existing client pipeline becomes the proof point.

### Technical Shape
- **Astro front-end** (reuse marketing template) with a chat-like interface for plugin description.
- **Claude Sonnet** for the heavy lifting — takes the natural language description and generates the full plugin scaffold: index.ts, sandbox-entry.ts, Astro components, admin Block Kit UI, email templates, tests.
- **Cloudflare Worker** backend for orchestration, queue management, and build pipeline.
- **Plugin template engine** — our existing plugin code (EventDash, MemberShip) becomes the few-shot examples in the prompt. The AI learns from real production plugins.
- **Automated validation pipeline:** TypeScript compilation, ESLint, basic runtime smoke test in a sandboxed environment.
- **Key components:** Chat UI, prompt orchestration layer, code generation engine, validation pipeline, artifact storage (R2), plugin preview environment.

### One-Session Scope
**Ships in v1:**
- Web UI: describe your plugin, get generated code
- Generates: index.ts, sandbox-entry.ts, basic Astro component, README
- Validates: TypeScript compilation, ESLint pass
- Download as ZIP or push to GitHub repo
- 5 free generations, then paid
- Supports common patterns: content display, form handling, API integration, data storage

**V2 (follow-up sessions):**
- Admin UI generation (Block Kit)
- Stripe integration scaffolding
- Email template generation
- Automated marketplace submission
- Plugin editing/iteration via chat ("add a dark mode toggle")
- Plugin analytics (installs, usage)
- Team workspaces for agencies

### Unit Economics
- **Free tier:** 5 plugin generations (acquisition + activation)
- **Pay-per-plugin:** $29 for a basic plugin, $99 for complex (Stripe, auth, multi-component)
- **Agency plan:** $199/month — unlimited generations, priority queue, white-label
- **Margin:** ~70-80%. Sonnet is expensive per generation (~$0.50-2.00 in tokens for a full plugin), but the pricing absorbs this comfortably. At $29/plugin with $2 cost, that's 93% gross margin.

### Why Now
- Emdash's plugin ecosystem is nascent. There are maybe a dozen plugins total. The WordPress plugin directory has 60,000+. That gap is the opportunity.
- Code generation via LLMs just hit the quality threshold where generated plugins actually work. Six months ago, the output would have been too buggy. Today, with Sonnet-class models and our production examples as few-shot prompts, the quality bar is crossable.
- We have the only production Emdash plugins in existence (MemberShip, EventDash). Nobody else has the training data.
- The headless CMS market is consolidating. Emdash, Sanity, Payload — whoever has the best plugin ecosystem wins. We can be the kingmaker.
