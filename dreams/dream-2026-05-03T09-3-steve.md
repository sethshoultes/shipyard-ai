# DREAM Brainstorm — Steve (Design Quality, User Delight)

**Date:** 2026-05-03
**Persona:** Steve — taste-maker, pixel-perfectionist, believes invisible UX is the best UX
**Mandate:** 3 product ideas. Each must ship in one session, have a clear user, and a real distribution channel.

---

## Idea 1: Zenith

**One-liner:** A Gutenberg block that generates contextual featured images in a single click, feeling like it shipped with WordPress core.

**Why it delights:**
No settings panels. No dashboard bloat. A block that lives where the user already works — the editor. The generated image is automatically sized, optimized, and placed. The loading state is a subtle Gutenberg skeleton, not a modal. It feels like WordPress got an AI upgrade overnight.

**Dogfood angle:**
We publish frequently on our agency blog. Every post needs a featured image. Currently we hand-craft them or borrow from Unsplash. This replaces that friction with one click.

**Distribution:**
WordPress.org plugin repository. Target: content creators and agencies who care about post polish.

**One-session feasibility:** High. We know the Gutenberg block API, we have the Claude → Workers AI pipeline wired from Pinned and Dash. The block itself is ~200 lines of React.

---

## Idea 2: Pulse

**One-liner:** A living status page for your AI agency, embedded as a WordPress admin widget or standalone SaaS.

**Why it delights:**
Agents have personalities. Pulse gives each one an avatar, a color, and a heartbeat. When an agent is thinking, the card "breathes." When it finishes, it snaps green with a satisfying chime. The dashboard turns operations theater into ambient awareness. No more tab-switching to Cloudflare logs to see if a worker died.

**Dogfood angle:**
We run multi-agent cycles daily. Right now we tail logs like animals. Pulse would be the first thing we open each morning.

**Distribution:**
Open-source npm package + WordPress plugin. Agencies embed it; indie hackers run the SaaS version.

**One-session feasibility:** Medium-High. Cloudflare Workers already expose logs. We wire SSE from a Worker to a React frontend. The WP widget wrapper is trivial. The polish is in the CSS.

---

## Idea 3: Lumina

**One-liner:** Browser-based Remotion studio: paste a script, preview a video in seconds, render with TTS.

**Why it delights:**
Video tools are either iMovie-grade simple or After Effects-grade complex. Lumina is the sweet spot: a JSON config editor on the left, a live 720p canvas on the right, and a "narrate" button that calls our TTS pipeline. Changing a line of text updates the preview in <2s. It feels like Figma for video.

**Dogfood angle:**
We create product explainers for every launch. Lumina would become our default storyboard tool before we commit to a full render.

**Distribution:**
SaaS with generous free tier. Product Hunt launch. Agency customers get white-label exports.

**One-session feasibility:** Medium. Remotion Player runs in browser. The heavy lift is wiring the TTS preview (we can stream low-quality samples fast). Core preview loop is solid.

---

## Steve's Bottom Line
Zenith is the purest expression of our WP craft. Pulse solves our own pain most viscerally. Lumina is the most fun to build. If I had to pick one, Zenith ships fastest and makes the user smile first.
