## The First-5-Minutes Lens

*"Be thankful for what you have; you'll end up having more. If you concentrate on what you don't have, you will never, ever have enough."*

But users don't walk in grateful — they walk in skeptical. The first five minutes determine whether that skepticism turns to trust or abandonment.

## Shipyard AI (shipyard.company)

### First-5-Minutes Experience

**Entry Point:** shipyard.company

**Current Flow:**
1. Land on homepage — "PRD in. Production out."
2. See token-based pricing (500K-2M tokens)
3. Scroll through agent personas
4. See "Get Started" form
5. Submit project details
6. Wait for quote

**What's Improved Since Last Cycle:**
- 4 live example sites now exist (bellas, craft, dental, yoga)
- 7 plugins shipped — real proof of capability
- Theme marketplace operational

**Remaining Confusion Points:**

1. **Token pricing still alienates non-technical buyers**
   - "500K tokens" — What IS that?
   - Translation missing: "500K tokens ≈ 5-page marketing site"
   - Buyers making $5K decisions shouldn't need to understand API billing

2. **Example sites are buried**
   - Bellas, Craft, Dental, Yoga exist but...
   - Are they linked from homepage? Prominently?
   - "Built with Shipyard" showcase needs front-and-center placement

3. **Emdash dependency creates doubt**
   - "Emdash Sites" — What if I've never heard of Emdash?
   - Quick explainer missing: "Emdash is a modern CMS (like Squarespace, but faster)"
   - Platform lock-in concerns unanswered

4. **The wait destroys momentum**
   - "Quote within 24 hours" = "I'll forget I submitted this"
   - No instant acknowledgment of scope
   - No ballpark estimate while waiting

**Emotional State at 5 Minutes:** Impressed by concept, uncertain about fit. "This sounds powerful but I'm not sure it's for someone like me."

### Improvement Opportunities

- **Token translator widget** — Slider: "My project is approximately [5 pages / 20 pages / complex site]" → "That's roughly [X tokens / $Y]"
- **Portfolio hero section** — First thing visitors see: 4 live sites with screenshots, one click to view live
- **Instant scope estimate** — After form submission: "Based on your description, this looks like a Standard Site (~1M tokens). We'll confirm details within 24 hours."
- **Emdash one-liner** — "Built on Emdash — a lightning-fast CMS running on Cloudflare's edge network. [Learn more]"

## LocalGenius (localgenius.company)

### First-5-Minutes Experience

**Entry Point:** localgenius.company

**Current Flow:**
1. Land on homepage — "You didn't open a restaurant to do marketing"
2. See features: website, reviews, social, email
3. "5 minutes to setup" promise
4. Free trial CTA
5. Enter business name → auto-pull Google Business Profile

**What's Working:**
- "Virtual employee" framing is brilliant — relatable, non-technical
- Auto-import from Google is genuine magic
- Price transparency builds trust
- The demo phone mockup shows the conversational UI clearly

**Remaining Confusion Points:**

1. **Industry mismatch persists**
   - Demo shows "Maria's Kitchen" (restaurant)
   - Dentist visiting thinks: "Does this work for me?"
   - No industry selector: "See it for: Restaurant | Dental | Salon"

2. **Conversational UI is unfamiliar**
   - Most SMB owners expect dashboards, buttons, menus
   - "Talk to your marketing" — What do I say first?
   - No sample prompts visible on landing page

3. **"5 minutes to setup" doesn't clarify what happens AFTER**
   - Setup is fast, but then what?
   - First-time users stare at empty chat
   - No guided first task: "Let's start by responding to your latest review"

4. **Analytics Weekly Digest mentioned but not demonstrated**
   - One of the best features is invisible on landing page
   - Show a sample digest: "Here's what your week looked like"

**Emotional State at 5 Minutes:** Curious, cautiously optimistic. "I want this to work but I need to see proof it works for MY business."

### Improvement Opportunities

- **Industry toggle on landing page** — "I'm a: Restaurant | Dental | Salon | Retail | Other" — demo content adapts
- **First message is FROM LocalGenius** — System speaks first: "Welcome! I found your Google profile. You have 18 reviews with a 4.1 average. Ready to respond to your 2 unanswered reviews?"
- **Prompt suggestions visible** — "Try asking: 'Post about our weekend special' or 'Reply to my latest review'"
- **Sample Weekly Digest** — Show the actual email with real numbers: "This week: 3 new reviews, 2.1K social impressions..."

## Dash (WP Command Palette)

### First-5-Minutes Experience

**Entry Point:** WordPress plugin install

**Current Flow:**
1. Install plugin from WordPress directory
2. See admin notice: "Dash is ready — try Cmd+K!"
3. Press Cmd+K
4. Empty search bar appears
5. Type something... or stare blankly

**What's Working:**
- Zero configuration — works immediately
- Clean, minimal design
- Keyboard shortcut is universally understood

**Remaining Confusion Points:**

1. **Empty search bar = "Now what?"**
   - No placeholder hint
   - No suggestion of what to type
   - First-time users don't know commands exist

2. **Mode switching is secret knowledge**
   - `>` for commands, `@` for users
   - Where would anyone learn this?
   - Documentation exists but nobody reads it

3. **No celebration of success**
   - First successful navigation feels... flat
   - User saves time but doesn't feel it
   - No "You just did that in 2 seconds!"

**Emotional State at 5 Minutes:** Mild interest. "Oh, that's kind of useful." Habit unlikely to form.

### Improvement Opportunities

- **Dynamic placeholder** — Rotating text: "Search posts, pages, settings..." → "Type > for commands" → "Type @ to find users"
- **First-open onboarding** — One-time modal: "Here's what Dash can do: [3 examples with animations]"
- **Inline hints** — When search is empty, show: "Pro tip: Type > to see all available commands"
- **Success feedback** — Small animation or message after first command: "That took 2 seconds"

## Pinned (WP Sticky Notes)

### First-5-Minutes Experience

**Entry Point:** WordPress plugin install → Dashboard

**Current Flow:**
1. Install plugin
2. Dashboard widget appears
3. Empty board: "Double-click to add a note"
4. Create first note
5. Note appears in colorful sticky format

**What's Working:**
- Zero configuration
- Sticky note metaphor is universally understood
- Double-click to create is intuitive

**Remaining Confusion Points:**

1. **Empty board is lonely**
   - Just one instruction line
   - No warmth, no example notes
   - No "here's how teams use this"

2. **@mentions are invisible**
   - Critical feature, zero discoverability
   - First-time user would never try @username
   - Team value proposition hidden

3. **Solo user feels pointless**
   - "Why would I leave notes for myself?"
   - Personal use cases not communicated
   - Value prop assumes team context

**Emotional State at 5 Minutes:** Underwhelmed. "It's cute but I have sticky notes on my desk."

### Improvement Opportunities

- **Welcome note from Pinned** — Pre-populated: "Welcome! Try @mentioning a teammate, or pin this note to keep it at the top. Double-click anywhere to create your first note."
- **Sample notes visual** — Empty state shows 3-4 ghost notes in different colors, demonstrating what a active board looks like
- **Solo user hook** — "Flying solo? Use Pinned for todo reminders, client feedback notes, or content ideas."
- **Feature hint on hover** — When hovering near mention input: "Tip: Type @ to notify a teammate"

## Great Minds Plugin

### First-5-Minutes Experience

**Entry Point:** GitHub README → Claude Code installation

**Current Flow:**
1. Find repo on GitHub
2. Read README (2000+ words)
3. Run installation command
4. Create PRD file
5. Run pipeline command
6. Watch terminal output... and wait

**What's Working:**
- Clear installation instructions
- Slash command interface familiar to Claude Code users
- Comprehensive documentation exists

**Remaining Confusion Points:**

1. **README is a textbook, not an onboarding**
   - 14 agents, 17 commands, daemon modes, token budgets
   - New user drowning in information
   - Quick Start exists but is buried

2. **"Create a PRD" is intimidating**
   - What format? How detailed? What's required?
   - Template exists but discovery is unclear
   - Sample PRD not immediately visible

3. **No feedback during execution**
   - Pipeline runs but... what's happening?
   - How long will this take?
   - Is it working or stuck?

4. **No "Hello World" experience**
   - First project = real commitment
   - No 5-minute demo that shows magic
   - Trust before trial

**Emotional State at 5 Minutes:** Intimidated. "This looks powerful but I don't have time to figure it out."

### Improvement Opportunities

- **QUICKSTART.md** — Separate file: "Your first project in 5 minutes" with a trivial sample PRD
- **Sample PRD included** — A "hello world" PRD that builds a simple landing page in <5 minutes
- **Progress visualization** — Status output: "Phase 2/7: Building... (Feature 3 of 5 complete)"
- **Estimated time remaining** — "Approximate completion: 8 minutes"

## Top 3 First-5-Minutes Fixes (This Cycle)

### 1. Shipyard Token Translator + Portfolio Hero (CRITICAL)

**Problem:** Buyers don't understand token pricing; no proof of capability visible

**Solution:**
- Homepage hero: Grid of 4 live sites with screenshots + "View Live" buttons
- Token translator: Simple selector showing "This project type ≈ $X"
- Instant estimate after form: "Based on your input, this looks like..."

**Impact:** Reduces skepticism, accelerates purchase confidence

**Effort:** Low — content and UI changes, no new features

### 2. LocalGenius Industry Selector + System First Message (HIGH)

**Problem:** Demo shows restaurant; non-restaurant users doubt relevance

**Solution:**
- Landing page toggle: "I'm a: Restaurant | Dental | Salon | Other"
- Demo adapts to selection
- After signup, LocalGenius speaks first: "Welcome [Business]! I found your Google profile..."

**Impact:** Reduces industry-mismatch bounce, accelerates first action

**Effort:** Medium — requires demo content variants

### 3. Universal Empty State Overhaul (MEDIUM)

**Problem:** Dash and Pinned feel dead on first use

**Solution:**
- Dash: Rotating placeholder hints + first-open modal
- Pinned: Pre-populated welcome note + ghost notes visualization
- Both: Feature discovery hints ("Type > for commands")

**Impact:** Reduces abandonment, teaches features without documentation

**Effort:** Low — UI polish, no backend changes

## The 5-Minute Promise

Every product should answer these questions in under 5 minutes:

| Question | Should Be Answered By |
|----------|----------------------|
| "Is this for someone like me?" | Industry/role selector, relevant demo |
| "What can this do?" | Immediate example or preview |
| "How do I start?" | System-initiated first action |
| "Did I succeed?" | Clear feedback on first action |
| "Why should I come back?" | Teaser of next value (ranking, stats, etc.) |

Currently, no product answers all five. That's the work.
