## Executive Summary

I create shows that people binge, rewatch, and obsess over. The secret isn't just great first episodes—it's **the hook that pulls you back**. Looking at this portfolio, I see strong pilots but weak season arcs. Users might try these products, but **why would they come back tomorrow? Next week? Next month?**

Let me diagnose the retention gaps and prescribe the hooks that create habit.

---

## The Retention Framework

Great retention comes from three forces:

1. **Routine Integration** — Product becomes part of daily/weekly workflow
2. **Progress Visibility** — Users see their investment paying off over time
3. **Anticipation Creation** — Something to look forward to, something that pulls them back

Let's evaluate each product:

---

## Product-by-Product Retention Analysis

### 1. LocalGenius — The Weekly Digest Saves It

**Current Retention Hooks:**
- Weekly Monday digest with metrics
- Automated social posting (passive engagement)
- Review notifications (reactive engagement)

**Retention Score:** 6/10

**What's Working:**
The Monday digest is smart—it creates a weekly appointment. "Did my efforts work?" brings people back.

**What's Missing:**

| Gap | Impact | Fix |
|-----|--------|-----|
| No daily reason to open | Users forget product exists between digests | Add daily AI-generated insight notification |
| No streaks/milestones | No sense of progress | "You've been marketing consistently for 30 days!" |
| No competitive element | No external motivation | "Businesses like yours that post 3x/week see 47% more engagement" |
| No content calendar view | Can't see what's coming | Show next 7 days of planned content, let users feel in control |

**The Hook I'd Add:** **"Morning Briefing"**

Every morning at 8am local time, LocalGenius sends a 30-second digest:
- "3 new reviews overnight (4.7 avg)"
- "Today's post: [preview]"
- "One thing to consider: [AI insight]"

This creates **daily anticipation**. The user wakes up thinking, "What did LocalGenius find for me today?"

**Retention Target:** Daily active usage → Weekly digest opens → Monthly renewal

### 2. Shipyard AI — The One-Night Stand Problem

**Current Retention Hooks:**
- None visible

**Retention Score:** 2/10

**The Problem:** Shipyard is a transaction, not a relationship. User submits PRD → gets site → leaves forever.

This is the "one-night stand" of business models. Great revenue per encounter, but no relationship building.

**What's Missing:**

| Gap | Impact | Fix |
|-----|--------|-----|
| No post-launch relationship | Customer churns immediately | Maintenance tier (Buffett's point) |
| No "check back" reason | Site ships, user forgets Shipyard exists | Performance dashboard showing site metrics |
| No expansion path | User has no reason to buy more | "Your site is doing well—time to add a blog?" |

**The Hook I'd Add:** **"Site Performance Story"**

After launch, Shipyard sends monthly "Site Story" updates:
- "Your site had 2,847 visitors this month (+34%)"
- "Top performing page: /pricing (89 conversions)"
- "Lighthouse score: 94 (better than 87% of sites)"
- "Recommendation: Add testimonials section—sites like yours see 23% more conversions"

**Why This Works:**
- Creates ongoing relationship from one-time purchase
- Proves Shipyard's quality over time
- Opens door to upsells (new features, redesigns, second sites)
- The "recommendation" creates anticipation for next project

**Retention Target:** One-time buyer → Monthly site report opener → Repeat buyer within 12 months

### 3. Dash — Habit Formation Goldmine

**Current Retention Hooks:**
- Muscle memory (Cmd+K becomes instinctive)
- Recent items tracking (personalizes over time)
- Performance (faster than alternatives)

**Retention Score:** 7/10

**What's Working:**
Keyboard shortcuts create habit. Once someone learns Cmd+K, they can't go back to clicking through menus. This is **procedural retention**—the product becomes part of how they work.

**What's Missing:**

| Gap | Impact | Fix |
|-----|--------|-----|
| No usage insights | Users don't know they're benefiting | "You saved 47 minutes this week with Dash" |
| No progression system | Power users plateau | Unlock advanced commands as usage increases |
| No team features | Single-user only | Share custom commands with team |

**The Hook I'd Add:** **"Weekly Time Saved" Report**

Every Friday, Dash shows:
- "This week you used Dash 127 times"
- "Estimated time saved: 53 minutes"
- "Most used command: > New Post (34x)"
- "Pro tip: Try `/clear cache` to speed up your site"

**Why This Works:**
- Quantifies invisible value (time saved)
- Creates weekly anticipation (what's my score this week?)
- Introduces new features gradually (pro tips)
- Shareable ("I saved 3 hours this month with Dash")

**Retention Target:** Install → Daily use within 1 week → "Can't live without it" within 1 month

### 4. Pinned — The Team Dependency Hook

**Current Retention Hooks:**
- @mentions create notifications
- Acknowledgment system requires return visits
- Note expiration creates urgency

**Retention Score:** 5/10

**What's Working:**
The @mention system creates **social obligation**. If a teammate mentions you, you feel compelled to respond. This is powerful.

**What's Missing:**

| Gap | Impact | Fix |
|-----|--------|-----|
| No team activity feed | Can't see what others are doing | "Sarah archived 3 notes, Tom created 2 new notes" |
| No note analytics | Don't know what's effective | "Notes with @mentions get acknowledged 3x faster" |
| No templates for common workflows | Every note is from scratch | "Deploy checklist", "Client handoff", "Bug report" |
| No slack/email integration | Stays siloed in WP | Notify in the tools people already use |

**The Hook I'd Add:** **"Team Pulse" Dashboard**

A real-time view showing:
- Who's active on the board right now
- Recent note activity (created, archived, acknowledged)
- "Stale notes" that need attention
- Team responsiveness score ("Average acknowledgment time: 2.3 hours")

**Why This Works:**
- Creates ambient awareness of team activity
- Social proof encourages participation
- Gamifies responsiveness (nobody wants to be the slow acknowledger)
- Makes the dashboard a destination, not just a tool

**Retention Target:** Team install → Daily check-ins within 1 week → Required part of workflow within 1 month

### 5. Great Minds — The Learning Loop

**Current Retention Hooks:**
- Memory accumulation (agency gets smarter)
- Dream cycle consolidation
- Board reviews create learning moments

**Retention Score:** 6/10

**What's Working:**
The memory system is a retention flywheel. Every project makes the agency smarter, which makes the next project better, which builds confidence in the system.

**What's Missing:**

| Gap | Impact | Fix |
|-----|--------|-----|
| No visible memory growth | Users don't see intelligence accumulating | "Great Minds now has 187 memories (+32 this week)" |
| No "what we learned" summary | Insights are buried in logs | After each project: "3 new patterns discovered" |
| No comparative improvement | Can't see getting better | "This project had 43% fewer QA issues than your average" |

**The Hook I'd Add:** **"Agency Intelligence Report"**

Monthly report showing:
- "Total memories: 187 (+32 this month)"
- "Patterns learned: Component library structure, Test-first workflow preference"
- "Quality trend: QA pass rate improved from 78% to 94%"
- "Suggestion: Based on your projects, consider standardizing on TypeScript"

**Why This Works:**
- Visualizes the invisible (learning)
- Creates investment (the more you use it, the smarter it gets)
- Demonstrates improvement (proof the system is working)
- Generates anticipation (what will it learn next?)

**Retention Target:** First project → See improvement on second project → Trust builds → Wouldn't build without it

## Cross-Product Retention Opportunities

### 1. The Portfolio Dashboard

If a user has multiple products (Dash + Pinned + LocalGenius), create a unified "Portfolio Pulse":
- Single view of all product activity
- Cross-product insights ("Your team is 34% more productive since adding Dash")
- Combined value visualization ("Total time saved this month: 12 hours")

### 2. The Achievement System

Create achievements that span products:
- "Power User" — Used Dash 500+ times
- "Team Player" — Acknowledged 50 Pinned notes
- "Marketing Machine" — LocalGenius posted 100 times
- "Agency Owner" — Built 10 sites with Shipyard

**Why:** Achievements create collection mentality and cross-product exploration.

### 3. The Community Hook

None of these products have community features. Community creates:
- "What are others doing?" curiosity
- Social proof and validation
- User-generated tips and workflows
- Belonging (hardest to leave)

## The Narrative Truth

Great products tell a story. The user is the hero. The product is the guide.

**Current story:** "Here's a tool. Good luck."

**Better story:** "Every day, you're getting better. Here's proof. Here's what's next."

The missing element across all products is **visible progress**. Users don't see themselves improving. They don't see their investment paying off. They don't have a reason to come back and check.

Add the progress layer. Add the anticipation. Add the reason to return.

Then watch retention compound.
